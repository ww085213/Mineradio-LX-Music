# Mineradio 1.5.6.2 更新说明

## 启动与覆盖升级兼容

本次发布重点解决 1.5.6.1 覆盖原版、旧版或其他二创版本后卡在 Mineradio 启动画面，以及使用 Wallpaper Engine/桌面融合后重启卡死的问题。

- 不再复制其他版本的整套 Chromium Profile，只迁移经过白名单校验的 Mineradio 配置、音源和桌面壳设置。
- 当前版本配置优先；旧版本只补充缺失项，损坏的 JSON 会先原样隔离到 `%APPDATA%\Mineradio\recovery` 再继续启动。
- 首次升级会在页面脚本执行前备份并关闭旧配置残留的 `wallpaperMode` 与 `desktopLyrics` 原生窗口状态，播放列表、音量、音质和封面等用户数据保留。
- 启动看门狗使用真实启动哨兵。普通安全重载失败时，会备份 Chromium Local Storage，再清理并从白名单侧车配置恢复。
- 所有启动恢复操作都有超时和二级兜底，不会无限停留在 splash 或 `about:blank`。
- 渲染器上报首页就绪后即视为启动成功；图片、字体等非关键子资源超时不会停止健康页面或让主进程退出。
- 移除启动阶段对 Google Fonts 的联网硬依赖，离线和国内网络环境也能直接进入。
- 版本变化时只隔离可重建的 Cache、GPU、Shader 和 Session Storage；单项失败会在下次启动继续重试。

## Wallpaper Engine 与桌面融合

- 桌面融合启用改为事务流程：嵌入或交互层失败时强制解绑，恢复可聚焦、可点击的普通窗口。
- 壁纸开关使用 FIFO 与代次校验，快速“开/关/开”不会被旧请求反向覆盖；关闭失败会自动重试并提示。

## 本地歌单删除与界面

- 左侧歌单列表为本地自建歌单和本地文件夹歌单加入可见的“删”按钮；自建歌单详情页与资料库管理页也支持删除和批量删除。
- 删除本地歌单只移除 Mineradio 中的歌单记录，不删除硬盘音乐文件，也不清空当前播放队列。
- 文件夹路径、旧版路径键、索引和快照会同步清理。
- 删除与后台文件夹扫描同时发生时，以用户最后的删除或重新导入操作为准；已删除歌单不会在扫描结束或重启后复活。
- 资料库卡片的“歌曲数 · 来源”文字上移 3px，并提高字重和对比度，避免贴近卡片底边。
